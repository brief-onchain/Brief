# 安全策略

[English Version](SECURITY.md)

## 支持版本

安全修复默认只覆盖最新 `main` 分支。

| 版本 | 是否支持 |
| --- | --- |
| main（最新） | 是 |
| 历史快照 | 否 |

## 漏洞反馈方式

在官方修复前，请避免公开披露细节。

1. 若仓库启用了 GitHub Security Advisories，请使用私密漏洞通道。
2. 若暂未启用，请私下联系维护者，并提供：
- 漏洞类型与影响范围
- 受影响文件/接口
- PoC 或复现步骤
- 可选修复建议

## 响应时效目标

- 72 小时内确认收到
- 7 天内给出初步分级与处理结论
- 修复发布时间按漏洞严重度与可利用性决定

## 重点关注范围

- 后端 API 输入校验与注入类风险
- 前端 SSR / API 代理链路行为
- 密钥与环境隔离
- 第三方数据源的信任边界

## 负责任披露说明

欢迎善意、负责任的安全研究。
未经明确授权，请勿访问用户资金、私钥或生产系统。
